LAML with MzScheme 300 ... 362 on Unix (Linux and Solaris).

MzScheme 300+ works fine with LAML on Linux.

Concretely, I have tested LAML version 32 with PLT version 350 on Linux. All examples work.

MzScheme 300+ uses UniCode and UTF-8. It seems difficult (impossible?) to use MzScheme 300+ together with 8 bit character sets (such as iso-latin-1). Thus, you should only use MzScheme 300+ if you are ready for Unicode and UTF-8.

I have not yet converted all my text files (and CGI data files) to UTF-8, and therefore I still use MzScheme 209 on Linux together with LAML.
